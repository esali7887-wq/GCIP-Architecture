# 15. INTEGRATION MAP (Entegrasyon Haritası)

Bu belge, Global Construction Intelligence Platform'un (GCIP) sahadaki dağınık verileri toplamak, planlama ve finans sistemleriyle konuşmak için kullandığı tüm entegrasyon API'lerinin (Uygulama Programlama Arayüzleri) teknik haritasını tanımlar.

Platform, KOBİ'lerin bütçe ve altyapı kısıtlarına uygun olarak **"Çift Katmanlı Entegrasyon"** mimarisiyle kurgulanmıştır.

---

## 15.1. Katman 1: Standart Entegrasyonlar (Herkes İçin - KOBİ & Bağımsız Mod)
Bu entegrasyonlar GCIP bünyesinde standart olarak gelir, ek bir yazılım lisansı (SAP, Primavera vb.) veya BT altyapısı gerektirmez. KOBİ şantiyelerinde sistemin tek başına çalışmasını (standalone) sağlayan çekirdek yapıdır.

### 1. Omnichannel Saha İletişim Arayüzü
*   *WhatsApp Business API / Telegram API:* Formenlerden gelen ses kayıtları (Speech-to-Text Whisper API entegrasyonuyla) ve şantiye fotoğrafları sisteme aktarılır.
*   *SMS Gateway & Magic Link Engine:* Müşavirlerin ve denetçilerin e-posta veya telefona gelen tek kullanımlık şifresiz linklerle (JSON Web Token - JWT korumalı) onay vermesi sağlanır.
*   *MS Teams / Slack Webhooks:* Kritik alarmlar (İSG Stop-work, bütçe aşımı) yöneticilerin haberleşme kanallarına anlık bildirim olarak gönderilir.

### 2. Lojistik ve Devlet Portalları Entegrasyonu
*   **GİB E-İrsaliye / E-Fatura API (Gelir İdaresi Başkanlığı):** Şantiyeye giren malzemenin resmi e-irsaliyesi, plaka ve şoför TC kimlik bilgisiyle Gelir İdaresi portalı üzerinden otonom sorgulanır ve stok kaydına dönüştürülür.
*   **ÇŞİDB YDS (Yapı Denetim Sistemi) REST/SOAP API:** Saha beton döküm günleri, donatı kontrol vizeleri ve İSG yasal ihbarları resmi yapı denetim sistemi ile çift yönlü senkronize edilir. Denetçinin YDS üzerinde verdiği vize/onay durumları API üzerinden okunarak Neo4j üzerindeki `ProgressLog.inspection_status` özelliğine yazılır.
*   **TÜBİTAK KamuSM Zaman Damgası (RFC 3161) API:** Yasal claim dosyaları, kesin hakediş icmalleri (`:HakedisIcmal`) ve komisyon mutabakatları (`:JointCommissionNode`) SHA-256 bütünlük kontrolüyle KamuSM sunucularında kriptografik olarak imzalanır ve `(:TimestampSignature)` düğümüyle Neo4j graf modelinde değiştirilemez olarak kilitlenir.
*   **Hazır Beton Santralleri (GPS/NFC API):** Dökülecek betonun santralden çıkış saati ve mikserin yoldaki GPS konumu API üzerinden izlenerek rölantide bekleme maliyetleri ($L_{time}$) engellenir.

### 3. Dış Ortam ve Meteoroloji API Entegrasyonu
*   *OpenWeatherMap / AccuWeather APIs:* Şantiyenin coğrafi konumuna göre anlık yağış, rüzgar hızı ve sıcaklık tahminleri okunur. Rüzgar hızı 50 km/s üzerine çıktığı an kule vinç çalışması için [STOP-WORK] alarmı tetiklenir. Yağışlı günler süre uzatımı claim dosyasına yasal kanıt olarak eklenir.

### 4. CAD ve PDF Dosya Arayüzleri
*   *ezdxf & PyPDF2 / pdfplumber:* Yüklenen 2D DWG, DXF ve PDF çizimlerindeki çizgileri, katmanları (layers) ve metin etiketlerini okuyarak geometrik zeka motoruna ham veri hazırlar (BOQ üretimi için).

## 15.2. Katman 2: Kurumsal Entegrasyonlar (Tamamen Opsiyonel - Büyük Holdingler İçin)
Bu entegrasyonlar sadece Primavera P6 ve kurumsal ERP sistemleri kullanan büyük holdingler/projeler için opsiyonel birer eklenti (Plug-in) olarak devreye alınır. KOBİ'ler için bu modüller kapalıdır ve kurulmalarına gerek yoktur.

### 1. Primavera P6 ve CPM Planlama Entegrasyonu (Çift Yönlü)
*   *Teknik Altyapı:* Python tarafında Java tabanlı **MPXJ (Microsoft Project Exchange Java Wrapper)** kütüphanesi kullanılır.
*   *İçeri Aktarım (Import):* Yerel `.xer` veya XML dosyalarından WBS (İş Kırılım Yapısı), aktiviteler, hedef tarihler (Baselines), mantıksal öncül-ardıl ilişkileri (Predecessor/Successor) ve kaynak atamaları okunarak Neo4j grafik veritabanına aktarılır.
*   *Dışarı Aktarım (Export):* Sahada otonom doğrulanan günlük ilerleme ve süre uzatımı (EOT) kararları sonrası revize edilen CPM iş programı, Primavera uyumlu yeni bir `.xer` dosyası olarak otonom ihraç edilir.

### 2. Kurumsal ERP Sistemleri Entegrasyonu
*   *Teknik Altyapı:* RESTful API veya SOAP web servisleri üzerinden güvenli veri alışverişi.
*   *SAP ERP (S/4HANA):* RFC ve BAPI köprüleriyle satınalma siparişleri (Purchase Orders) okunur, onaylanan hakedişler için fatura girişleri otomatik tetiklenir.
*   *Oracle Fusion ERP / Microsoft Dynamics 365:* Cari hesap verileri okunur. Hakediş onaylandığı an banka ödeme talimatı ve hakediş muhasebe fişleri otonom oluşturulur.

### 3. Autodesk ACC (BIM 360) APIs
*   *İşlev:* IFC ve Revit modellerindeki 3D metraj verilerini ve tasarım çakışmalarını (Clash Detection) okumak için bulut API entegrasyonu sağlar.
