# 21. Ajan Orkestrasyon Yapısı (Orchestrator Architecture)

Bu belge, Global Construction Intelligence Platform'un (GCIP) çoklu yapay zeka ajanları (Multi-Agent Swarm) arasındaki iş koordinasyonunu, oturum bazlı ortak durum yönetimini (Shared State), anlamsal yönlendirmeyi (Semantic Routing) ve insan onay döngülerini (Human-in-the-loop) yöneten **ORC-1 (Orkestratör Ajan)** motorunun teknik mimarisini, güvenlik bariyerlerini ve arayüz protokollerini tanımlar.

---

## 21.1. Orkestrasyon Mimari Konsepti ve Yerleşimi

ORC-1, sisteme gelen tüm yapılandırılmamış verileri (ses kayıtları, PDF sözleşmeler, irsaliyeler, CAD çizimleri) ve API tetikleyicilerini karşılayan **merkezi denetleyicidir**. Ajanlar monolitik bir LLM çağrısıyla değil, LangGraph tabanlı durum grafikleri (State Graphs) üzerinden yönetilir.

```
                         Girdi (Ses, PDF, CAD, API)
                                    │
                                    ▼
                        ┌───────────────────────┐
                        │  ORC-1 ORKESTRATÖR   │
                        │  (Semantic Router)    │
                        └───────────┬───────────┘
                                    │
           ┌────────────────────────┼────────────────────────┐
           ▼                        ▼                        ▼
     [Asenkron Çalışma]       [Senkron Çalışma]       [Paralel Çalışma]
     LOG-8 (Lojistik)         SCH-3 (Planlama/CPM)    SAF-7 (İSG Fotoğraf)
     CON-5 (Sözleşme/Claim)   BIL-4 (Hakediş)         QS-2 (CAD Metraj)
           │                        │                        │
           └────────────────────────┼────────────────────────┘
                                    ▼
                         ┌─────────────────────┐
                         │    Shared State     │
                         │ (Redis / Neo4j Graf)│
                         └──────────┬──────────┘
                                    │ (Düşük Güven Skoru / Kritik Karar)
                                    ▼
                         ┌─────────────────────┐
                         │ SPE-9 WhatsApp Loop │
                         │  (İnsan Onay Kapısı)│
                         └─────────────────────┘
```

## 21.2. Çapraz Belge Entegrasyon Protokolü (1 - 20)

ORC-1 Orkestratörünün mimari bütünlüğü sağlamak amacıyla diğer 20 belge ile olan teknik entegrasyon noktaları aşağıda tanımlanmıştır:

### 21.2.1. [1. VISION.md](file:///D:/CAI%20--/1. VISION.md) (Shadow Mode ve Geri Alma)
- **Shadow Mode:** ORC-1, sisteme ilk 90 gün boyunca gelen verileri işlerken `SharedState.shadow_mode` değerini `True` olarak işaretler. Veritabanına doğrudan onaylı kayıt atmaz; tüm işlemleri izleme amaçlı simüle eder.
- **Rollback (Geri Alma):** Hatalı veri veya insan müdahalesiyle yapılan düzeltmelerde ORC-1, Neo4j üzerindeki değişiklikleri soft-delete yaparak bir önceki geçerli baseline sürümüne geri döner.

### 21.2.2. [2. PROBLEMS.md](file:///D:/CAI%20--/2. PROBLEMS.md) (Şantiye Direncini Aşma)
- **WhatsApp Ses Arayüzü:** KOBİ şantiyelerindeki teknik adaptasyon sınırlarını aşmak için orkestratörün birincil tetikleyicisi `SPE-9` üzerinden Whisper API ile çözümlenmiş sesli komutlardır.

### 21.2.3. [3. RISK_TAXONOMY.md](file:///D:/CAI%20--/3. RISK_TAXONOMY.md) (Risk Seviyeleri ve Otomatik Blokaj)
- **Alarm Seviyeleri:** ORC-1, `RSK-6` tarafından hesaplanan risk seviyesini dinler. L4 (Kritik) seviyesinde bir risk (Örn: Nakit akış sıkışması veya kritik İSG ihlali) tespit edildiğinde, hakediş ödeme akışını dondurur.

### 21.2.4. [4. ENGINEERING_KNOWLEDGE_BASE.md](file:///D:/CAI%20--/4. ENGINEERING_KNOWLEDGE_BASE.md) (RAG ve Kaynak Doğrulama)
- **Kanıt Sunumu (XAI):** Orkestratör, `CON-5` (Sözleşme) veya `SCH-3` (Planlama) ajanlarından bir karar çıktısı aldığında, bu kararı destekleyen teknik şartname veya mevzuat maddelerini `gcip_specs_vector_index` üzerinden sorgulayarak çıktıya ekler.

### 21.2.5. [5. USER_PERSONAS.md](file:///D:/CAI%20--/5. USER_PERSONAS.md) (Kullanıcı Rolüne Duyarlı Çıktı Modeli)
- **Arayüz Yönetimi:** ORC-1, durum nesnesindeki `user_role` parametresine göre çıktı dilini ve biçimini ayarlar. Formenler için sade WhatsApp butonları, mühendisler için teknik Cypher logları üretir.

### 21.2.6. [6. COMPETITOR_ANALYSIS.md](file:///D:/CAI%20--/6. COMPETITOR_ANALYSIS.md) (Performans Logları ve Veri Ağı Moat'u)
- **Taşeron Değerlendirme:** Tamamlanan her iş kaleminin sonunda orkestratör `LOG-8` (fire), `SAF-7` (İSG) ve `SCH-3` (gecikme) verilerini birleştirerek taşeron performans karnesini Neo4j `(:PerformanceEvaluation)` düğümüne kaydeder.

### 21.2.7. [7. ARCHITECTURE.md](file:///D:/CAI%20--/7. ARCHITECTURE.md) (Offline-First ve Çakışma Çözümü)
- **Senkronizasyon:** Çevrimdışı şantiyelerde RxDB üzerinde lokal çalışan orkestratör, internet bağlantısı sağlandığında merkezi sunucu ile yerel durum nesnelerini (Shared State) `7. ARCHITECTURE.md` kurallarına göre senkronize eder.

### 21.2.8. [8. PRODUCT_PRINCIPLES.md](file:///D:/CAI%20--/8. PRODUCT_PRINCIPLES.md) (Halüsinasyon Engelleme ve Kanıt Bütünlüğü)

## 21.3. Semantik Yönlendirme (Semantic Routing & Intent Classification)

Sahadan veya dış servislerden gelen verilerin türünü ve hangi ajana gitmesi gerektiğini belirlemek için milisaniyeler düzeyinde çalışan vektör tabanlı bir **Semantic Router** katmanı kullanılır. Bu katman, gereksiz büyük model (LLM) çağrılarını engelleyerek token tasarrufu sağlar.

### Semantik Router Yapılandırması (Python)

```python
from semantic_router import Route
from semantic_router.encoders import HuggingFaceEncoder
from semantic_router.layer import RouteLayer

# 1. Uzman Ajan Rotaları ve Örnek İfadeler
quantity_survey_route = Route(
    name="quantity_surveyor",
    utterances=[
        "Bu çizimdeki temel beton metrajını hesapla",
        "Kat planındaki duvar alanını bul",
        "Kolon demir miktarlarını tablodan çıkar",
        "CAD dosyasındaki keşif pozu miktarlarını karşılaştır"
    ]
)

schedule_planning_route = Route(
    name="scheduling_agent",
    utterances=[
        "İş programını 10 gün öteye kaydır",
        "Primavera kritik yol analizini çalıştır",
        "Süre uzatımı kararını iş programına yansıt",
        "CPM aktivitesinin float süresini hesapla"
    ]
)

billing_payment_route = Route(
    name="billing_agent",
    utterances=[
        "Bu ayın taşeron hakedişini hazırla",
        "İSG cezalarını hakedişten düş",
        "Avans kesintisini yansıt",
        "Hakediş icmal tablosunu çıkar"
    ]
)

# 2. Vektör Kodlayıcı Kurulumu (Token Tasarrufu için Küçük Yerel Model)
encoder = HuggingFaceEncoder(name="sentence-transformers/all-MiniLM-L6-v2")
routes = [quantity_survey_route, schedule_planning_route, billing_payment_route]
rl = RouteLayer(encoder=encoder, routes=routes)

def route_intent(user_query: str) -> str:
    """Girdinin hangi ajana gideceğini cosine similarity ile belirler."""
    decision = rl(user_query)
    return decision.name if decision.name else "orchestrator_fallback"
```

## 21.4. Üretim Düzeyi Güvenlik Katmanları (Protective Shields)

Orkestratör, şantiye sahasının fiziksel ve altyapısal zorlukları altında sistemin kontrolden çıkmasını önlemek için 4 otomatik koruma kalkanı barındırır:

### 21.4.1. Sonsuz Döngü Koruması (Runaway Guard)
Ajanların birbirini zincirleme ve döngüsel olarak tetiklemesini (Ajan A $\rightarrow$ Ajan B $\rightarrow$ Ajan A) durdurmak amacıyla her işlem zinciri bir `current_hop_count` sayacı ile takip edilir. Bu değer `max_hops = 5` limitine ulaştığında orkestratör yürütmeyi keser ve durumu insan onayına sevk eder.

### 21.4.2. Bulut / Yerel LLM Fallback (Disaster Recovery)
Orkestratör, bulut LLM servislerine (örn. OpenAI/Anthropic API'leri) erişim krizlerinde veya gecikme süresi 3.0 saniyeyi aştığında, şantiyedeki yerel sunucuda barındırılan **Llama-3-8B-Instruct** (yerel çıkarım motoru) sistemine otomatik olarak failover (geçiş) yapar.

### 21.4.3. İşlem Kurtarma ve Sıcak Başlangıç (Hot Recovery)
Olası elektrik veya bağlantı kesintilerinde, yarım kalan iş akışlarının kaybolmaması için Redis üzerindeki `SharedState` her durum değişiminde diske yazılır (Redis AOF). Sunucu yeniden başladığında yarım kalan `task_id` üzerinden akışı otomatik olarak kaldığı düğümden (node) sürdürür.

### 21.4.4. Ajan Bazlı Bütçe Limiti (Token Runaway Guard)
Kontrol dışı token harcamalarının KOBİ bütçesine zarar vermesini engellemek amacıyla tek bir işlem zincirinin (sorgu $\rightarrow$ ajanlar $\rightarrow$ çıktı) maksimum maliyeti **0.50$ (USD)** ile sınırlandırılmıştır. Limit aşıldığında sistem işlemi dondurur.

## 21.5. Paylaşılan Durum Yönetimi (Shared State & Session Memory)

Ajanlar arası veri paylaşımı ve oturum kontrolü için kullanılan veri yapıları:

### Pydantic Durum Şemaları (Orchestrator State Schema)

```python
from pydantic import BaseModel, Field
from typing import Dict, List, Optional
from uuid import UUID, uuid4
from datetime import datetime

class ActiveAgentTask(BaseModel):
    task_id: UUID = Field(default_factory=uuid4)
    agent_name: str  # QS-2, SCH-3, BIL-4 vb.
    status: str      # PENDING, RUNNING, COMPLETED, FAILED
    started_at: datetime = Field(default_factory=datetime.utcnow)
    input_payload: Dict
    output_payload: Optional[Dict] = None

class SharedState(BaseModel):
    session_id: UUID = Field(default_factory=uuid4)
    tenant_id: str = Field(..., description="Çoklu kiracı veri izolasyon anahtarı")
    project_id: str
    user_role: str = Field(..., description="Kullanıcı rolü (5. Madde Personalar ile uyum)")
    active_wbs_activity_id: Optional[str] = None
    active_subcontractor_id: Optional[str] = None
    shadow_mode: bool = True  # Canlı yazma kısıtlaması (1. ve 17. Madde Uyum)
    
    # 1. Sonsuz Döngü Koruması
    current_hop_count: int = Field(0, description="Döngü engelleyici adım sayacı")
    max_hops: int = Field(5, description="Maksimum izin verilen ardışık ajan tetikleme limiti")
    
    # 4. Maliyet Bütçe Kontrolü
    accumulated_cost_usd: float = Field(0.0, description="Bu işlem zincirinde biriken token maliyeti")
    max_token_budget_usd: float = Field(0.50, description="İşlem başına maksimum harcama limiti")
    
    # Ajanların ortak kullandığı veri havuzu
    context_data: Dict = Field(default_factory=dict)
    
    # Doğrulama Eşikleri ve İnsan Onayı (Human-in-the-loop)
    confidence_score: float = 1.0
    pending_human_approval: bool = False
    approval_token: Optional[str] = None
    
    # İzlenebilirlik Logları (8. Madde XAI Uyum)
    task_history: List[ActiveAgentTask] = []
    explainability_nodes: List[str] = Field(default_factory=list, description="Referans alınan Neo4j Düğüm ID'leri")
```

## 21.6. Olay Güdümlü Orkestrasyon Akışı (LangGraph DAG)

Ajanlar arasındaki iş akışları, LangGraph tabanlı dinamik yönlendirilmiş devirsiz grafik (DAG) motoruyla yönetilir. 

### LangGraph İş Akışı Yönetimi (Python)

```python
from langgraph.graph import StateGraph, END
from typing import Dict

# 1. Ajan Düğüm Fonksiyonları
def call_logistics_node(state: SharedState) -> Dict:
    """LOG-8 Ajanını tetikler ve irsaliye verilerini durum nesnesine yazar."""
    task = ActiveAgentTask(agent_name="LOG-8", input_payload={"project_id": state.project_id})
    
    # Simüle edilmiş LOG-8 çıktısı (İrsaliyedeki hazır beton miktarının tespiti)
    logistics_output = {
        "material_type": "C30_Concrete",
        "declared_qty": 50.0,
        "verified_qty": 48.5,
        "variance_pct": 0.03
    }
    
    return {
        "context_data": {**state.context_data, "logistics": logistics_output},
        "task_history": state.task_history + [task],
        "current_hop_count": state.current_hop_count + 1
    }

def call_risk_node(state: SharedState) -> Dict:
    """RSK-6 Ajanını tetikler ve sapmaları analiz eder."""
    task = ActiveAgentTask(agent_name="RSK-6", input_payload=state.context_data)
    
    logistics_data = state.context_data.get("logistics", {})
    variance = logistics_data.get("variance_pct", 0.0)
    
    # Risk Sınıflandırması (3. Madde Risk Taksonomisi Uyum)
    risk_level = "GREEN" if variance < 0.02 else "ORANGE" if variance < 0.05 else "RED"
    
    return {
        "context_data": {**state.context_data, "risk_level": risk_level},
        "task_history": state.task_history + [task],
        "current_hop_count": state.current_hop_count + 1
    }

def call_billing_node(state: SharedState) -> Dict:
    """BIL-4 Ajanını tetikler, eğer risk RED ise ödemeyi bloke eder."""
    task = ActiveAgentTask(agent_name="BIL-4", input_payload=state.context_data)
    
    risk = state.context_data.get("risk_level")
    if risk == "RED" or state.current_hop_count >= state.max_hops:
        return {
            "pending_human_approval": True,
            "confidence_score": 0.65,
            "context_data": {**state.context_data, "billing_status": "BLOCKED"},
            "task_history": state.task_history + [task],
            "current_hop_count": state.current_hop_count + 1
        }
    
    return {
        "pending_human_approval": False,
        "context_data": {**state.context_data, "billing_status": "APPROVED_FOR_PAYMENT"},
        "task_history": state.task_history + [task],
        "current_hop_count": state.current_hop_count + 1
    }

# 2. Grafik Yapılandırması ve Akış Yönetimi
workflow = StateGraph(SharedState)

# Düğümlerin Tanımlanması
workflow.add_node("logistics_agent", call_logistics_node)
workflow.add_node("risk_agent", call_risk_node)
workflow.add_node("billing_agent", call_billing_node)

# Kenarların (Edges) Bağlanması
workflow.set_entry_point("logistics_agent")
workflow.add_edge("logistics_agent", "risk_agent")

# Koşullu Geçiş Mantığı (Conditional Routing)
def check_risk_and_route(state: SharedState):
    # Döngü koruması kontrolü
    if state.current_hop_count >= state.max_hops:
        return "billing_agent"
    if state.context_data.get("risk_level") in ["ORANGE", "RED"]:
        return "billing_agent"
    return END

workflow.add_conditional_edges(
    "risk_agent",
    check_risk_and_route,
    {
        "billing_agent": "billing_agent",
        END: END
    }
)
workflow.add_edge("billing_agent", END)

orchestrator_app = workflow.compile()
```

## 21.7. İnsan Onay Döngüsü (Human-in-the-loop) ve Fallback Arayüzü

Elde edilen sonuçların güvenilirlik skoru ($C_{\text{score}}$) düştüğünde ya da risk seviyesi L4 olarak belirlendiğinde orkestratör işlemleri dondurur ve **insan teyidi** mekanizmasını tetikler.

### Doğrulama Eşikleri ve Aksiyon Planı
1. **$C_{\text{score}} \ge 0.90$:** Otonom onay. İşlemler doğrudan veritabanına yazılır.
2. **$0.75 \le C_{\text{score}} < 0.90$:** Denetimli onay. İşlemler veritabanına yazılır ancak "Mühendis Onayı Bekliyor" olarak işaretlenir.
3. **$C_{\text{score}} < 0.75$:** Blokaj. İşlem dondurulur, `SPE-9` üzerinden şantiye sorumlusuna WhatsApp etkileşimli onay mesajı gönderilir.

## 21.8. LLM Format Sapmalarını Engelleme (Pydantic Parser & Auto-Retry)

Orkestratörün alt ajanlardan gelen verileri doğru şekilde işleyebilmesi için yapılandırılmış (structured) çıktılara ihtiyacı vardır. LLM'lerin şema dışına çıkmasını önlemek için **Pydantic Output Parser** ve otomatik yeniden deneme (retry) kural dizisi uygulanır.

```python
from langchain_core.output_parsers import PydanticOutputParser
from langchain_core.prompts import PromptTemplate
from langchain_core.exceptions import OutputParserException
from langchain_openai import ChatOpenAI

# Ajan Yönlendirme Karar Şeması
class RoutingDecision(BaseModel):
    intent: str
    target_agent: str
    confidence: float
    required_parameters: Dict[str, str]

parser = PydanticOutputParser(pydantic_object=RoutingDecision)

prompt = PromptTemplate(
    template="Kullanıcı girdisini analiz et.\n{format_instructions}\nGirdi: {query}\n",
    input_variables=["query"],
    partial_variables={"format_instructions": parser.get_format_instructions()}
)

def execute_structured_routing(user_query: str, max_retries: int = 3) -> RoutingDecision:
    model = ChatOpenAI(model="gpt-4o-mini", temperature=0.0)
    chain = prompt | model | parser
    
    retries = 0
    while retries < max_retries:
        try:
            decision: RoutingDecision = chain.invoke({"query": user_query})
            return decision
        except OutputParserException as e:
            # Hata durumunda modeli hatayla birlikte uyararak tekrar dene
            retries += 1
            if retries == max_retries:
                # Son hakta da başarısız olunursa güvenli fallback modunu çalıştır
                return RoutingDecision(
                    intent="unknown",
                    target_agent="SPE-9",
                    confidence=0.0,
                    required_parameters={"error": str(e)}
                )
```
