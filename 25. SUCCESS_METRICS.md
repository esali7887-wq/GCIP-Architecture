## 25.2. Metrik Tanımları ve Formülleri

### 25.2.1. Finansal ve Kaçak Geri Kazanım Metrikleri

#### 1. Kaçak Geri Kazanım Oranı (Leakage Recovery Rate - $R_{\text{recovery}}$)
Şantiyedeki hatalı hakedişler, unutulan avanslar ve fazla malzeme sarfiyatından kurtarılan toplam finansal değerin, toplam proje bütçesine oranıdır:

$$R_{\text{recovery}} = \frac{\sum L_{\text{advance}} + \sum L_{\text{payment}} + \sum L_{\text{waste}}}{\text{Toplam Proje Bütçesi}}$$

*   $L_{\text{advance}}$: Hakedişte mahsup edilmesi unutulan avanslar.
*   $L_{\text{payment}}$: Fazla veya hatalı beyan edilip mühendis/AI tarafından elenen haksız hakediş ödemeleri.
*   $L_{\text{waste}}$: Depodan sahaya kontrolsüz çıkarılan ve engellenen malzeme zayiatı tutarı.

#### 2. Talep Kabul Başarı Oranı (Claim Acceptance Rate - $R_{\text{claim\_accept}}$)
`CON-5` ajanı tarafından otonom hazırlanan claim (süre uzatımı/maliyet) taleplerinin idarece onaylanan oranını ölçer:

$$R_{\text{claim\_accept}} = \frac{\text{Onaylanan Ek Süre (Gün) veya Tutar (USD)}}{\text{Talep Edilen Ek Süre (Gün) veya Tutar (USD)}}$$

*Hedef:* $\ge 0.80$ (Talep edilen hakların en az %80'inin idarece kabul edilmesi).

#### 3. Kazanılmış Değer Analizi (Earned Value Management - EVM)
Projenin bütçe ve zaman uyumunu izlemek için kullanılan standart metrikler:
- **Maliyet Performans Endeksi (Cost Performance Index - CPI):**
  $$\text{CPI} = \frac{\text{Earned Value (EV)}}{\text{Actual Cost (AC)}}$$
  *Hedef:* $\ge 1.00$ (Bütçe aşımı yok).
- **Program Performans Endeksi (Schedule Performance Index - SPI):**
  $$\text{SPI} = \frac{\text{Earned Value (EV)}}{\text{Planned Value (PV)}}$$
  *Hedef:* $\ge 1.00$ (Proje zamanında ilerliyor).

---

### 25.2.2. Operasyonel ve Yapay Zeka Doğruluk Metrikleri

#### 1. Metraj Doğruluk Oranı (Quantity Survey Precision - $P_{\text{metraj}}$)
`QS-2` ajanı tarafından 2D CAD dosyalarından çıkarılan otonom keşif metrajları ile şantiyede fiilen gerçekleşen as-built metrajlar arasındaki doğruluk oranıdır:

$$P_{\text{metraj}} = 1.0 - \frac{\text{abs}(\text{Teorik Metraj} - \text{As-Built Metraj})}{\text{Teorik Metraj}}$$

*Hedef:* $\ge 0.96$ (En fazla %4 sapma).

#### 2. Whisper Ses Çözümleme Başarı Oranı ($C_{\text{whisper}}$)
`SPE-9` asistanına şantiyeden gelen gürültülü ve şiveli ses kayıtlarının, ek teyit buton döngüsüne girmeden tek seferde doğru çözümlenme oranıdır:

$$C_{\text{whisper}} = \frac{\text{Tek Seferde Doğrulanan Ses Komutları}}{\text{Toplam Gelen Ses Komutu Sayısı}}$$

*Hedef:* $\ge 0.85$.

#### 3. YOLOv8 İSG Tespit Başarısı (HSE Safety Precision/Recall)
Kameralardan gelen görüntülerde baret, yelek ve emniyet kemeri ihlallerinin otonom tespit doğruluğudur:
- **Precision (Kesinlik):** Yanlış alarm üretmeme başarısı ($\ge 0.92$).
- **Recall (Duyarlılık):** Fiili ihlallerin kaçırılmama başarısı ($\ge 0.95$).

---

### 25.2.3. İSG ve Kalite Uyum Metrikleri (HSE & QA/QC)

#### 1. İSG İhlal Kapatma Hızı ($T_{\text{hse}}$)
`SAF-7` tarafından tespit edilen İSG ihlallerinin, şantiye yönetimi tarafından sahada giderilip kapatılma süresidir:

$$T_{\text{hse}} = \text{İhlal Kapatılma Zamanı} - \text{İhlal Tespit Zamanı}$$

*Hedef:* L4 (Kritik) ihlaller için $< 15 \text{ dakika}$.

#### 2. Uyuşmazlık Çözüm Süresi (Dispute Resolution Velocity - $T_{\text{dispute\_res}}$)
Şantiyede imalat veya cezalardan dolayı tetiklenen uyuşmazlıkların (`:DisputeNode`), komisyon arayüzü (`23. DASHBOARDS.md` uyumlu) üzerinden çözüme kavuşturulma hızıdır:

$$T_{\text{dispute\_res}} = \text{Komisyon Karar Onay Tarihi} - \text{Dispute Kilit Tarihi}$$

*Hedef:* $< 48 \text{ saat}$ (Mutabakat krizlerinin hızlıca aşılması).

---

### 25.2.4. Kullanıcı Benimseme ve Aktivite Metrikleri (Adoption & Engagement)

#### 1. WhatsApp Formen Aktivite Oranı
Şantiyedeki aktif formenlerin günlük olarak sesli mesaj veya görsel irsaliye yükleme yoluyla sistemle kurduğu etkileşim sıklığı:
- **DAU / MAU Oranı:** Günlük Aktif Kullanıcı / Aylık Aktif Kullanıcı oranı.
- *Hedef:* $\ge 0.70$ (Haftada en az 5 gün aktif katılım).

#### 2. Şantiye Onboarding Süresi ($T_{\text{onboarding}}$)
Yeni bir projenin `24. ONBOARDING.md` adımları doğrultusunda sisteme kaydedilmesinden, ilk WBS doğrulama ve Shadow Mode aktivasyonuna kadar geçen süredir.
- *Hedef:* $< 24 \text{ saat}$ (take-over projeler için $< 36 \text{ saat}$).

## 25.3. Metrik Hesaplama Motoru (Python)

Orkestratör, her hakediş dönemi sonunda aşağıdaki metrik hesaplama motorunu koşturarak projelerin genel sağlık puanını ve ROI verilerini hesaplar:

```python
from pydantic import BaseModel, Field
from typing import Dict
import numpy as np

class ProjectMetricsInput(BaseModel):
    total_budget: float
    planned_value: float
    actual_cost: float
    earned_value: float
    leakage_advance_usd: float
    leakage_payment_usd: float
    leakage_waste_usd: float
    total_safety_violations: int
    resolved_safety_violations: int
    total_disputes_count: int
    resolved_disputes_duration_hours: float
    total_claims_submitted_usd: float
    total_claims_approved_usd: float

class MetricsCalculator:
    def __init__(self, data: ProjectMetricsInput):
        self.data = data

    def calculate_evm(self) -> Dict[str, float]:
        """EVM CPI ve SPI metriklerini hesaplar."""
        cpi = self.data.earned_value / self.data.actual_cost if self.data.actual_cost > 0 else 1.0
        spi = self.data.earned_value / self.data.planned_value if self.data.planned_value > 0 else 1.0
        return {"cpi": round(cpi, 2), "spi": round(spi, 2)}

    def calculate_leakage_recovery(self) -> Dict[str, float]:
        """Finansal kaçak kurtarma oranını ve toplam geri kazanımı hesaplar."""
        total_recovered = (
            self.data.leakage_advance_usd + 
            self.data.leakage_payment_usd + 
            self.data.leakage_waste_usd
        )
        recovery_rate = total_recovered / self.data.total_budget if self.data.total_budget > 0 else 0.0
        return {
            "total_recovered_usd": total_recovered,
            "recovery_rate": round(recovery_rate, 4)
        }

    def calculate_claim_success(self) -> float:
        """Hak talebi kabul başarı oranını hesaplar."""
        if self.data.total_claims_submitted_usd == 0:
            return 1.00
        return round(self.data.total_claims_approved_usd / self.data.total_claims_submitted_usd, 2)

    def calculate_dispute_resolution_speed(self) -> float:
        """Uyuşmazlık başına ortalama çözüm süresini saat cinsinden hesaplar."""
        if self.data.total_disputes_count == 0:
            return 0.0
        return round(self.data.resolved_disputes_duration_hours / self.data.total_disputes_count, 1)

    def calculate_hse_compliance(self) -> float:
        """İSG ihlal kapatma başarı oranını hesaplar."""
        if self.data.total_safety_violations == 0:
            return 1.00
        compliance = self.data.resolved_safety_violations / self.data.total_safety_violations
        return round(compliance, 2)

    def get_project_health_score(self) -> float:
        """Finansal, zaman ve İSG metriklerini birleştirerek 100 üzerinden sağlık skoru verir."""
        evm = self.calculate_evm()
        hse = self.calculate_hse_compliance()
        claim_success = self.calculate_claim_success()
        
        # Ağırlık katsayıları
        w_cpi = 0.30
        w_spi = 0.30
        w_hse = 0.20
        w_claim = 0.20
        
        # Normalize edilmiş değerler (Maksimum 100 puan)
        cpi_score = min(100.0, evm["cpi"] * 100.0)
        spi_score = min(100.0, evm["spi"] * 100.0)
        hse_score = hse * 100.0
        claim_score = claim_success * 100.0
        
        health_score = (cpi_score * w_cpi) + (spi_score * w_spi) + (hse_score * w_hse) + (claim_score * w_claim)
        return round(health_score, 1)
```
